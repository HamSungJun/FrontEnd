    BEM 이란?

        - CSS 유지 보수와 , 코드 간결함을 위해 존재하는 방법론
        - Block : Element : Modifier 의 약자.

            #Block 

                - 웹페이지 상에서 자기 스스로 의미를 갖거나 독립적으로 존재하는 개체

            #Element 

                - 블록 컴포넌트 내부의 원소를 이루는 개체들

            #Modifier 

                - 블록 , 블록 내부 원소들의 상태나 , 행동을 나타내는 플래그


    장점 

        - 블록 컴포넌트는 내부 원소에 영향을 미치지 않는 독립적인 스타일을 적용한다. (모듈화)
        - 각기 다른 독립적인 블록 컴포넌트는 페이지 내에서 재사용이 가능해진다. (재사용성)
        - 코드 구조가 견고해지면서 이해가 용이하다.(가독성)

    네이밍

        #공통 - 아이디 , 태그네임은 되도록 이용하지 않고 클래스 위주로 작성한다.

        #BLOCK

            - 독립적으로 존재 할 수 있는 작명을 한다.
            - CSS 속성은 다른 BLOCK 이나 ELEMENT에 비 의존성을 갖는다.

        #ELEMENT

            - 블록 내부의 원소이며 독립적인 의미를 갖지 않는다.
            - 요소들 간에는 의미적으로 동일한 속성을 갖는다.
            - (블록 클래스__엘리먼트 이름);

        #MODIFIER

            - 블록 혹은 엘리먼트의 상태를 나타내도록 작성한다.
            - 오리지널 클래스명을 유지하면서 MOD 클래스 명을 추가 기입하는 식으로 이용한다.
            - (블록 클래스--상태자 이름)
            - (블록 클래스__엘리먼트 이름--상태자 이름);
            - (블록 클래스__엘리먼트 이름--상태자 이름-상태자 이름2)
